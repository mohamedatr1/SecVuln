# Web Form Inspector & Parser

### Description
This script is a specialized web auditing tool designed to detect and extract HTML forms from a webpage. It is used to map out the "Attack Surface" of a website by identifying where user input is accepted, which is the first step in testing for vulnerabilities like SQL Injection or XSS.



### Technical Workflow
* **Headless Browser Execution**: Launches a Chromium instance in the background for efficient DOM parsing without the overhead of a graphical interface.
* **Page Navigation**: Directs the browser to the target URL to load all static and dynamic elements, ensuring that forms generated by JavaScript are also captured.
* **Form Discovery**: Uses the `query_selector_all("form")` method to locate every `<form>` tag within the page's Document Object Model (DOM).
* **HTML Extraction**: Iterates through each discovered form and extracts its internal HTML code (`inner_html`). This reveals critical details such as:
    * **Input Names**: Used to identify parameter names for fuzzing.
    * **Input Types**: Identifies hidden fields, password fields, and text areas.
    * **Action Endpoints**: Shows where the data is being sent for processing.
* **Clean Termination**: Closes the browser instance to free up system resources after the extraction is complete.

### Requirements
* **Playwright**: For high-level browser automation and DOM interaction.
* **Chromium**: The browser engine used to parse the web content.
